2.2.4. A large integer by Karatsuba multiplication¶

After the post [VC], here we explore another large integer

"EssentialsObjectTest, protocol tests"
testInspectLargeInteger

   | large n |
   n := 42 factorial.
   large := n dcMultiplyInteger: n.

   self assert: large equals: n * n.

   ^ self exportSlotsGraphOf: large

where the message

"Integer, protocol *Containers-Essentials"
dcMultiplyInteger: anInteger

   | base |
   base := 10.

   ^ ((self between: 0 and: base - 1) and: [ 
         anInteger between: 0 and: base - 1 ])
        ifTrue: [ self * anInteger ]
        ifFalse: [ 
           | a b bDigits |
           (self numberOfDigitsInBase: base)
           > (anInteger numberOfDigitsInBase: base)
              ifTrue: [ 
                 a := self.
                 b := anInteger ]
              ifFalse: [ 
                 a := anInteger.
                 b := self ].

           bDigits := b numberOfDigitsInBase: base.

           ^ a
                halves: [ :xHigh :xLow :xn :xl | 
                   b
                      halves: [ :yHigh :yLow :yn :yl | 
                         | p pp xyHigh xyLow |
                         p := xHigh + xLow dcMultiplyInteger: yHigh + yLow.
                         xyHigh := xHigh dcMultiplyInteger: yHigh.
                         xyLow := xLow dcMultiplyInteger: yLow.

                         pp := p - xyHigh - xyLow * (base ** xl).
                         xyHigh * (base ** xl double) + pp + xyLow ]
                      at: xl
                      digits: bDigits
                      base: base ]
                base: base ]

implements the algorithm described in [KT05], at page 232. Such algorithm runs in \(O(n^{\log_{2}{3}})\) because the input numbers \(x\) and \(y\) are broken in two parts

\[\begin{split}x = x_{1}\cdot 10^{{{n}\over{2}}} + x_{0} \\ y = y_{1}\cdot 10^{{{n}\over{2}}} + y_{0}\end{split}\]

respectively, and there are three recursive #dcMultiplyInteger: message sends. The implementation follows from both the fact

\[x\,y = x_{1}\,y_{1}\cdot 10^{n} + (x_{1}\,y_{0} + x_{0}\,y_{1})\cdot 10^{{{n}\over{2}}} + x_{0}\,y_{0}\]

and

\[(x_{1} + x_{0})\,(y_{1} + y_{0}) = x_{1}\,y_{1} + \left(x_{1}\,y_{0} + x_{0}\,y_{1}\right) + x_{0}\,y_{0}\]

respectively, more references can also be found in [Wik]. Two auxiliary messages

"Integer, protocol *Containers-Essentials"
halves: aBlock base: aBase

   | n |
   n := self numberOfDigitsInBase: aBase.

   ^ self
        halves: aBlock
        at: (n / 2) ceiling
        digits: n
        base: aBase

and

"Integer, protocol *Containers-Essentials"
halves: aBlock at: l digits: n base: aBase

   | low high lSucc |
   lSucc := l + 1.

   low := (1 to: l) inject: 0 into: [ :acc :i | 
             | d |
             d := (self digitAt: i base: aBase) * (aBase ** (i - 1)).
             d + acc ].

   high := (lSucc to: n) inject: 0 into: [ :acc :i | 
              | d |
              d := (self digitAt: i base: aBase) * (aBase ** (i - lSucc)).
              d + acc ].

   ^ aBlock
        value: high
        value: low
        value: n
        value: l

helps the recursive message.

Some fractions kiss each other, here we see kisses by mediants via the polymorphism of the message #\/,

"Fraction, protocol *Containers-Essentials"
\/ anObject

   ^ anObject mediantFraction: self

"Integer, protocol *Containers-Essentials"
\/ anObject

   ^ anObject mediantInteger: self

that both implementations dispatch back to their arguments according to

"Integer, protocol *Containers-Essentials"
mediantFraction: aFraction

   ^ aFraction numerator + self / (aFraction denominator + self)

"Fraction, protocol *Containers-Essentials"
mediantFraction: aFraction

   ^ aFraction numerator + self numerator
     / (aFraction denominator + self denominator)

for the former and to

"Integer, protocol *Containers-Essentials"
mediantInteger: anInteger

   ^ self + anInteger / 2

"Fraction, protocol *Containers-Essentials"
mediantInteger: anInteger

   ^ self numerator + anInteger numerator
     / (self denominator + anInteger denominator)

"BlockClosure, protocol *Containers-Essentials"
kissingFractions

   | fractions news level |
   level := 1.
   fractions := SortedCollection with: 0 with: 1.

   [ 
   news := OrderedCollection new.
   fractions overlappingPairsDo: [ :a :b | 
      | f |
      f := a \/ b.
      (self
          cull: f
          cull: fractions
          cull: news
          cull: level) ifTrue: [ news add: f ] ].
   news isEmpty ] whileFalse: [ 
      fractions addAll: news.
      level := level + 1 ].

   ^ fractions
        removeFirst;
        removeLast;
        asArray

2.3.6. Binary Reflected Gray Codes¶

In [Gra53], Frank Gray introduces “an ordering of the binary numeral system such that two successive values differ in only one bit” – from Wikipedia,

"EssentialsObjectTest, protocol tests"
testInspectBRGCodes

   ^ self exportSlotsGraphOf:
        ((0 to: 15) collect: [ :each | each asShapeBRGCDots ] as: RSGroup)

also known as [Sloa]. The message

"Integer, protocol *Containers-Essentials-Visualizations"
asShapeBRGCDots

   ^ self asShapeBinaryDots: (self bitBRGC printStringBase: 2) asArray

relies on both

"Integer, protocol *Containers-Essentials"
bitBRGC

   ^ self bitXor: self >> 1

that computes the Gray representation corresponding to the receiver Integer, and

"Integer, protocol *Containers-Essentials-Visualizations"
asShapeBinaryDots: repr

   | dots |
   dots := repr collect: [ :each | 
              | int |
              int := Integer readFrom: each asString.
              RSCircle new
                 model: int;
                 radius: Float goldenRatio double double;
                 color: Color white;
                 withBorder;
                 yourself: [ :c | 
                    c color: (int
                              ifZero: [ Color white ]
                              ifOne: [ c border color ]
                              ifTwo: [ 
                                 c border
                                    color: Color black;
                                    color ]
                              otherwise: [ Error signal ]) ] ].

   RSHorizontalLineLayout new
      gapSize: Float goldenRatio;
      on: dots.

   ^ RSComposite new
        model: repr;
        shapes: dots;
        yourself

that computes the dots-oriented representation, empty dots stand for 0s while full dots stand for 1s.
